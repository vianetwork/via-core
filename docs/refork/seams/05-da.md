# Seam 05 — Data availability (dispatcher + clients)

## What via patched in v24

- `core/lib/via_da_clients`: own Celestia client + `fallback/` wrapper + HTTP client +
  external-node variant, with its own wiring layer (856 added LOC).
- `core/lib/via_da_dispatcher` (+ `via_da_dispatcher_lib`): fork of `da_dispatcher`,
  delta only **+65/−287**; dispatches pubdata blobs and records inclusion in the
  via-added `via_data_availability` table (migration `20240916094925`, extended by
  `20251030121202_via_da_chunk_blob` for chunked blobs).
- DA inclusion data flows into inscriptions (btc_sender) and the verifier's
  `via_da_client`/`via_withdrawal_client`.

## The v29 extension point

- `core/lib/da_client` defines `pub trait DataAvailabilityClient` with
  `dispatch_blob`, **`ensure_finality`** (new since v24), and `get_inclusion_data`,
  plus a `node/` wiring module behind the `node_framework` feature.
- **v29 ships a native Celestia client**: `core/node/da_clients/src/celestia/`
  (with generated protos), configured through
  `core/lib/config/src/configs/da_client/celestia.rs` and a `CelestiaSecrets` entry.
- `core/node/da_dispatcher` exists with its own node layers.

## Port approach

1. **Evaluate replacing via's Celestia client with upstream's** before porting
   `via_da_clients`. Concretely: diff capabilities (auth modes, namespace handling,
   blob size limits, commitment format returned as `inclusion_data`) between
   `core/lib/via_da_clients/src/celestia/client.rs` and the pin's
   `core/node/da_clients/src/celestia/client.rs`. The DA *commitment bytes* end up in
   inscriptions, so format compatibility is consensus-relevant for the new chain —
   regenesis means we may adopt upstream's format freely, but the differential eval
   must then compare semantics, not bytes.
2. Port the `fallback` wrapper as a thin `DataAvailabilityClient` decorator crate
   (`via_da_clients` shrinks to the fallback + any via-specific behavior).
3. Re-apply the small `via_da_dispatcher` delta onto v29 `da_dispatcher`; implement
   `ensure_finality` (new trait method) against Celestia finality.
4. Keep the `via_data_availability` tables (seam 06) unless v29's
   `data_availability` table now covers chunking — check during the unit.
